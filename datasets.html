Overview

LSM and ESM are reproducible noisy-speech benchmarks designed to separate noise-family and speech-style shifts. LSM (LibriSpeech-MUSAN) pairs read speech with out-of-family MUSAN noise, providing a controlled noise-family shift. ESM (EARS-MUSAN) retains the MUSAN noise source and the same mixing protocol but replaces LibriSpeech with expressive EARS speech. Comparing ESM with LSM therefore isolates the additional effect of a speech-style shift while holding the noise protocol fixed.

Each pair provides an aligned clean reference and its noisy mixture. SNR levels are assigned round-robin by pair index.

Construction protocol

Parallel deterministic scripts implement the same construction protocol and differ only in the speech source. For each pair, speech and noise are resampled to 16 kHz. The noise is looped to the speech duration, shifted by a seeded offset, and scaled to the target SNR. A common rescaling factor then prevents clipping while preserving sample alignment between the clean reference and noisy mixture. Every random draw comes from a seeded generator (seed 1234 for LSM, seed 4321 for ESM). Regeneration from the public source corpora reproduces the released waveforms, and the published SHA-256 lists verify the resulting files.

Download & reproduce

The data can be obtained in two ways. The lightweight route regenerates the waveforms from the public source corpora, while the packaged route downloads the released mixtures directly.

Regenerate from source (bit-exact)

  • Build scriptsbuild_lsm.pybuild_esm.py
  • Per-mixture manifestslsm_manifest.csvesm_manifest.csv
  • SHA-256 checksumslsm_sha256.txtesm_sha256.txt
  • Source corpora LibriSpeech MUSAN EARS

Download the source corpora, run each build script, and check the output against the SHA-256 lists. The manifests record the index, SNR, file names and per-clip checksums of the released set.

Packaged mixtures

  • LSM (clean + noisy)lsm_ood.zipCC BY 4.0
  • ESM (clean + noisy)esm_ood.zipCC BY-NC 4.0

Packaged archives are attached to the repository release. Each archive holds clean/ and noisy/ at 16 kHz PCM16, named {lsm,esm}_INDEX_snrSNR.wav.
